# User Preferences

- When using Superpowers skills or workflows, never create commits automatically.
- Keep related changes bundled as one uncommitted work set by default.
- Only create a commit if the user explicitly asks for one.
- Never write or update tests unless the user explicitly asks to do so.
- When showing a plan, always include explicit file operations with exact paths for `Create`, `Update`, and `Delete`, using `none` when a category is empty.
